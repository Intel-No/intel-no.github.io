内网模拟企业网站：数据库主从复制 + 高可用集群部署

在完成三台 CentOS 7 虚拟机的主机名，网络解析，时间同步与基础防火墙配置之后，接下来需要正式安装数据库服务（如果没安装，点我查看基础篇章）

整体的网络拓扑如下：

检查三台机器是否已有 MariaDB / MySQL 残留

修改目的：
先确认系统里有没有旧版本数据库，避免后面安装 MariaDB 10.5 时冲突

首先先检查是否有数据库残留（在三台机器都执行）

 rpm -qa | grep -E "mariadb|mysql|MariaDB"

如果什么都不输出，说明很干净，可以继续

如果输出类似：

那么就先卸载旧版 MariaDB

systemctl stop mariadb 2>/dev/null
yum remove -y mariadb-server mariadb

然后再检查一次rpm -qa | grep -E "mariadb|mysql|MariaDB"

如果只剩类似这些：

mariadb-libs-5.5.68-1.el7.x86_64
php-mysqlnd-5.4.16-48.el7.x86_64

mariadb-libs 和 php-mysqlnd 可以先保留，这个是 CentOS 7 系统常见依赖，先留着就行。等安装 MariaDB 10.5 时，yum 会自动处理依赖关系

三台机器写入 MariaDB 10.5 官方 yum 源

现在三台都执行下面这段

cat > /etc/yum.repos.d/MariaDB.repo <<'EOF'
[mariadb]
name=MariaDB 10.5 Repository
baseurl=http://yum.mariadb.org/10.5/centos7-amd64
gpgkey=https://yum.mariadb.org/RPM-GPG-KEY-MariaDB
gpgcheck=1
enabled=1
EOF

然后三台都检查：

cat /etc/yum.repos.d/MariaDB.repo
